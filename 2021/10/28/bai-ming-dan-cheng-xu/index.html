白名单程序

为什么有的应用程序不需要提示UAC呢？

这是因为有的可以程序可以autoElevate(自动提升)

具有autoElevate属性为True的应用程序会在启动时自动提升权限，而这些应用程序往往都具备微软的签名，微软认为它是可信的。

这类程序被统称为白名单程序，故此，在该程序启动时，将会以管理员身份启动，就不会提示UAC

挖掘白名单程序

四点要求

1.system32目录下

2.程序的manifest标识的配置属性 autoElevate 为true

3. 程序不弹出UAC弹窗

4. 从注册表里查询Shell\Open\command键值对

脚本

import os 
from subprocess import * 
 
path = 'c:\windows\system32' 
files = os.listdir(path) 
print(files) 
def GetFileList(path, fileList): 
    newDir = path 
    if os.path.isfile(path): 
        if path[-4:] == '.exe': 
            fileList.append(path) 
    elif os.path.isdir(path): 
        try: 
            for s in os.listdir(path): 
                newDir=os.path.join(path,s) 
                GetFileList(newDir, fileList) 
        except Exception as e: 
            pass 
    return fileList 
files = GetFileList(path, []) 
print(files) 
for eachFile in files: 
    if eachFile[-4:] == '.exe': 
        command = r'.\sigcheck64.exe -m {} | findstr auto'.format(eachFile) 
        print(command) 
        p1 = Popen(command, shell=True, stdin=PIPE, stdout=PIPE) 
        if '<autoElevate>true</autoElevate>' in p1.stdout.read().decode('gb2312'): 
            copy_command = r'copy {} .\success'.format(eachFile) 
            Popen(copy_command, shell=True, stdin=PIPE, stdout=PIPE) 
            print('[+] {}'.format(eachFile)) 
            with open('success.txt', 'at') as f: 
                f.writelines('{}\n'.format(eachFile))

添加过滤

注：通常以shell\open\command命名的键值对存储的是可执行文件的路径

经过排查

启动FodHelper.exe时，它会检查是否存在以下注册表项：

HKCU:\Software\Classes\ms-settings\Shell\Open\command

Result显示的是

NAME NOT FOUND

那么我手动创建一个

继续

发现它还会去

HKCU:\Software\Classes\ms-settings\shell\open\command\DelegateExecute

那么我这里创建一个

字符串值:DelegateExecute

把默认键值对指向其他一个程序进行尝试

C:\Users\Administrator\Desktop\ProcessExplorer\procexp64.exe

当我运行c:\windows\system32\ComputerDefaults.exe的时候，发现不再弹出的是默认进程的界面，而是打开了procexp64.exe

整理思路

1.创建注册表

2.添加DelegateExecute这个键值对

3.修改默认数据指向exe路径

4.隐蔽性考虑

注：c++ 操作注册表如果遇到名称为”(默认)”的处理方法：用””（空串）来处理即可

API的调用，具体可以参考这里：https://docs.microsoft.com/zh-cn/windows/win32/com/registering-components

RegCreateKeyEx

RegCreateKeyEx是创建注册表项的函数

LONG RegCreateKeyEx(
  HKEY hKey,                                  // handle to open key
  LPCTSTR lpSubKey,                           // subkey name
  DWORD Reserved,                             // reserved
  LPTSTR lpClass,                             // class string
  DWORD dwOptions,                            // special options
  REGSAM samDesired,                          // desired security access
  LPSECURITY_ATTRIBUTES lpSecurityAttributes, // inheritance
  PHKEY phkResult,                            // key handle 
  LPDWORD lpdwDisposition                     // disposition value buffer
);
函数分析
hkey:根键盘的句柄

lpSubKey:子键的名字

Reserved:保留参数,必须为0

lpClass:该键的用户定义类类型,可以忽略此参数,此参数可以为NULL

dwOptions:有几个值,不适用置0即可,使用的时候具体查询

samDesired:指定要创建的密钥的访问权限的掩码

lpSecurityAttributes:指向SECURITY_ATTRIBUTES结构的指针

phkResult:传出参数,新创建或打开的子键的句柄

lpdwDisposition:有这个子键就-打开,没有就-创建

RegSetValueExA

RegSetValueEx是修改注册表项的函数

我们需要将它指向我们的exe路径

LONG RegSetValueEx(
  HKEY hKey,           // handle to key
  LPCTSTR lpValueName, // value name
  DWORD Reserved,      // reserved
  DWORD dwType,        // value type
  CONST BYTE *lpData,  // value data
  DWORD cbData         // size of value data
);
函数分析
hkey:根键的句柄

lpValueName:要设置的键值项

Reserved:保留参数,必须为0

dwType:lpData要设置的键值类型

lpData:要设置的键值数据

cbData:lpData参数指向的信息的大小,以字节为单位

SHDeleteKey

我们为了隐蔽性的考虑，最好是删除这个路径，那么就需要用到这个api

它可以删除一个注册表键，包括其下面的子键

LONG WINAPI SHDeleteKey (
  in HKEY hKey,
  in LPCTSTR lpSubKey
);
函数分析
hkey:根键的句柄,必须要有DELETE权限

lpSubKey:子键名
